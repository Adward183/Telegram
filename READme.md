<b>Описание проекта</b>:
Бот предназначен для обработки Excel файлов и автоматического создания отчетов по различным типам данных образовательного учреждения. Пользователи загружают Excel файлы (.xls или .xlsx), а бот анализирует данные и генерирует текстовые отчеты.

<b>Основные возможности</b>

<b>Типы поддерживаемых отчетов:</b>

<b>Расписание</b> - анализ недельного расписания занятий

<b>Темы уроков</b> - проверка корректности оформления тем уроков

<b>Оценки студентов</b> - анализ успеваемости студентов

<b>Посещаемость преподавателей</b> - мониторинг посещаемости преподавателями

<b>ДЗ преподавателей</b> - проверка выполнения домашних заданий преподавателями

<b>ДЗ студентов</b> - анализ сдачи домашних заданий студентами

<b>Особенности работы:</b>

Автоматическое определение типа отчета по структуре файла

Поддержка форматов .xls и .xlsx

Генерация текстовых отчетов в удобном формате

Сохранение истории обработанных файлов

<b>Использование бота</b>

<b>Команды для пользователей:</b>

<b>/start</b> - Начать работу с ботом, показывает главное меню

<b>/mystats</b> - Показать статистику пользователя (количество отчетов)

<b>/cancel</b> - Отменить текущее действие

<b>/id</b> - Показать ваш Telegram ID

<b>Команда для администратора:</b>

<b>/admin</b> - Показать админ-панель со статистикой (доступно только для ADMIN_IDS)

<b>Процесс работы:</b>

Отправьте команду /start

Выберите тип отчета из меню

Загрузите Excel файл соответствующего формата

Получите сгенерированный отчет в текстовом файле

<b>Форматы входных данных</b>

<b>Расписание:</b>

Файл должен содержать столбцы с днями недели (понедельник, вторник и т.д.)

<b>Темы уроков:</b>

Файл должен содержать столбец "Тема урока" с форматом: "Урок №X. Тема: Название темы"

<b>Оценки студентов:</b>

Файл должен содержать столбцы: FIO (ФИО), Homework (Домашняя работа), Classroom (Классная работа)

<b>Посещаемость преподавателей:</b>

Файл должен содержать столбцы: "ФИО преподавателя", "Посещаемость" или "Средняя посещаемость"

<b>ДЗ преподавателей:</b>

Файл должен содержать столбцы: "ФИО преподавателя", "проверено", "получено"

<b>ДЗ студентов:</b>

Файл должен содержать столбцы: FIO, "Percentage Homework" или "Homework Percentage"

<b>База данных:</b>

Бот использует SQLite базу данных для хранения статистики:

Таблица users: хранит информацию о пользователях (user_id, имя, количество отчетов)

Таблица reports: хранит историю обработки файлов

Базу данных можно просматривать и редактировать с помощью SQLite Browser.

<b>Модули проекта</b>

<b>Основные модули обработки:</b>

<b>excelProcessor.py</b> - главный модуль обработки Excel файлов, определяет тип отчета и вызывает соответствующий обработчик.

<b>reportSchedule.py</b> - генерация отчета по расписанию, подсчет количества пар по предметам.

<b>reportTopics.py</b> - проверка корректности оформления тем уроков.

<b>reportStudents.py</b> - анализ оценок студентов, выявление проблемных показателей.

<b>reportAttendanceTeachers.py</b> - анализ посещаемости преподавателей.

<b>reportHomeworkTeachers.py</b> - анализ проверки домашних заданий преподавателями.

<b>reportHomeworkStudents.py</b> - анализ выполнения домашних заданий студентами.

<b>Вспомогательные модули:</b>

<b>helpers.py</b> - функции для отправки отчетов и работы с файлами.

<b>Логика работы:</b>

Пользователь выбирает тип отчета через инлайн-кнопки

Пользователь загружает Excel файл

Бот определяет структуру файла (заголовки, типы данных)

В зависимости от выбранного типа вызывается соответствующий обработчик

Результат обработки сохраняется в текстовый файл и отправляется пользователю

Статистика использования сохраняется в базу данных

<b>Обработка ошибок:</b>

Проверка формата файла (.xls или .xlsx)

Проверка размера файла (до 30 МБ)

Проверка наличия необходимых столбцов в файле

Обработка пустых файлов и некорректных данных

<b>Основные библиотеки:</b>

pyTelegramBotAPI - работа с Telegram Bot API

pandas/openpyxl - обработка и чтение Excel файлов

SQLite - хранение статистики
